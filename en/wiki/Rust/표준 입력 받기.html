표준 입력 받기

Rust에서 터미널 입력은 보통 표준 입력(standard input)에서 읽는다. 핵심은 입력 자체가 문자열로 들어오고, 그 문자열을 필요한 타입으로 다시 해석한다는 점이다. 그래서 입력 코드는 보통 세 단계로 나뉜다.

  1. 표준 입력에서 문자열을 읽는다.
  2. 줄 끝 개행이나 공백을 정리한다.
  3. parse로 원하는 타입으로 바꾼다.

한 줄을 읽는 기본 형태​

가장 단순한 입력은 std::io::stdin().read_line()으로 받는다.

use std::io;

fn main() {
    let mut line = String::new();

    io::stdin()
        .read_line(&mut line)
        .expect("failed to read line");

    let line = line.trim();
    println!("input: {line}");
}

여기서 line은 String이어야 하고, read_line에는 &mut line을 넘긴다. 입력을 담을 문자열을 함수가 직접 수정해야 하므로 가변 참조가 필요하다.

read_line은 기존 문자열을 덮어쓰지 않고 뒤에 덧붙인다. 그래서 같은 버퍼를 반복해서 쓸 때는 읽기 전에 clear()로 비워야 한다.

또 하나 중요한 점은 엔터까지 입력으로 들어온다는 것이다. 사용자가 42를 입력하고 엔터를 누르면 문자열은 보통 "42\n"이 된다. Windows에서는 "42\r\n"이 될 수 있다. 그래서 숫자로 바꾸기 전에는 보통 trim()으로 앞뒤 공백과 개행을 제거한다.

한 줄에서 여러 숫자 읽기​

알고리즘 풀이를 처음 할 때는 아래처럼 한 줄을 읽고, 그 줄을 공백 기준으로 나눈 뒤 숫자로 바꾸는 형태를 많이 쓴다.

use std::io;

fn main() {
    let mut input = String::new();

    io::stdin()
        .read_line(&mut input)
        .unwrap();

    let nums: Vec<i32> = input
        .split_whitespace()
        .map(|x| x.parse().unwrap())
        .collect();

    println!("{nums:?}");
}

이 코드는 입력이 한 줄에 모두 들어온다고 가정한다.

3 5

이 경우 read_line이 한 줄을 읽고, split_whitespace가 "3"과 "5"를 분리한다. 이후 parse가 각각을 i32로 바꾸고, collect가 Vec<i32>로 모은다.

반대로 입력이 여러 줄로 들어오면 이 코드는 첫 줄만 읽는다.

3
5

이 경우 nums에는 첫 번째 값만 들어간다. 두 값을 모두 읽었다고 가정하고 이후 코드에서 두 번째 값을 꺼내면 문제가 생길 수 있다. 그래서 "한 줄 입력"인지 "전체 입력"인지가 중요하다.

입력은 실패할 수 있으므로 Result를 처리한다​

read_line은 Result<usize>를 반환한다. 성공하면 읽은 바이트 수가 나오고, 실패하면 I/O 오류가 나온다. 짧은 연습 코드에서는 expect로 실패 시 바로 중단해도 되지만, 함수 전체가 실패를 반환하는 구조라면 ?를 쓰는 편이 자연스럽다.

use std::error::Error;
use std::io;

fn main() -> Result<(), Box<dyn Error>> {
    let mut line = String::new();
    io::stdin().read_line(&mut line)?;

    let value: i32 = line.trim().parse()?;
    println!("{}", value * 2);

    Ok(())
}

이 코드에서는 입력 실패와 파싱 실패가 모두 바깥으로 전파된다. 연습용 알고리즘 풀이처럼 입력이 항상 정상이라고 가정하는 환경에서는 unwrap이나 expect를 쓰기도 하지만, 실제 프로그램에서는 실패를 어디에서 처리할지 먼저 정하는 편이 좋다.

여러 번 한 줄씩 읽기​

한 줄씩 반복해서 읽을 때는 버퍼를 재사용할 수 있다.

use std::io;

fn main() -> io::Result<()> {
    let stdin = io::stdin();
    let mut line = String::new();

    loop {
        line.clear();

        let bytes = stdin.read_line(&mut line)?;
        if bytes == 0 {
            break;
        }

        println!("line: {}", line.trim());
    }

    Ok(())
}

read_line이 Ok(0)을 반환하면 더 읽을 입력이 없다는 뜻이다. 파일이나 파이프에서 입력을 받을 때는 EOF를 만나면 이 상태가 된다.

줄 단위 반복이 목적이라면 lines()도 쓸 수 있다.

use std::io::{self, BufRead};

fn main() -> io::Result<()> {
    let stdin = io::stdin();

    for line in stdin.lock().lines() {
        let line = line?;
        println!("line: {line}");
    }

    Ok(())
}

stdin.lock()은 표준 입력을 잠근 뒤 BufRead 메서드를 쓸 수 있는 핸들을 만든다. 반복적으로 줄을 읽는 코드에서는 매번 새로 표준 입력을 잡는 것보다 이 형태가 의도가 분명하다.

공백 기준으로 여러 값을 읽기​

알고리즘 문제처럼 입력 전체를 한 번에 받은 뒤 공백 기준으로 나눠 쓰는 경우가 많다. 이때는 read_to_string으로 표준 입력 전체를 읽고, split_whitespace로 토큰을 만든다.

use std::error::Error;
use std::io::{self, Read};

fn main() -> Result<(), Box<dyn Error>> {
    let mut input = String::new();
    io::stdin().read_to_string(&mut input)?;

    let mut tokens = input.split_whitespace();

    let count: usize = tokens.next().expect("missing count").parse()?;
    let values: Vec<i64> = tokens
        .take(count)
        .map(str::parse)
        .collect::<Result<_, _>>()?;

    let sum: i64 = values.iter().sum();
    println!("{sum}");

    Ok(())
}

이 방식은 입력이 파일이나 온라인 저지처럼 끝이 정해진 스트림일 때 잘 맞는다. 반대로 사람이 터미널에서 직접 입력하는 대화형 프로그램에서는 EOF가 들어오기 전까지 기다릴 수 있으므로, 한 줄씩 읽는 방식이 더 자연스럽다.

타입 변환은 입력과 별개로 본다​

표준 입력에서 읽은 값은 처음에는 문자열이다. 숫자, 불리언, 열거형 같은 값으로 쓰려면 별도로 변환해야 한다.

let n: i32 = line.trim().parse()?;

parse는 목표 타입을 알아야 한다. 왼쪽 변수 타입으로 알려줄 수도 있고, 메서드에 직접 타입을 붙일 수도 있다.

let n = line.trim().parse::<i32>()?;

여러 토큰을 읽는 코드에서는 이 타입 명시가 더 중요하다. parse()만 써서는 컴파일러가 어떤 숫자 타입으로 바꿔야 하는지 모를 수 있다.

정리​

  • 한 줄 입력은 io::stdin().read_line(&mut line)으로 받는다.
  • read_line은 기존 String 뒤에 덧붙이고, 개행 문자도 포함한다.
  • 첫 줄에 필요한 값이 모두 있으면 read_line과 split_whitespace를 같이 쓸 수 있다.
  • 같은 버퍼를 반복해서 쓰려면 읽기 전에 clear()를 호출한다.
  • 숫자로 바꾸기 전에는 보통 trim().parse()를 쓴다.
  • 줄 단위 반복에는 stdin.lock().lines()가 읽기 쉽다.
  • 입력 전체를 한 번에 읽어 공백 기준으로 처리하려면 read_to_string과 split_whitespace를 조합한다.
  • read_to_string은 EOF까지 기다리므로 대화형 입력에는 조심해서 써야 한다.
