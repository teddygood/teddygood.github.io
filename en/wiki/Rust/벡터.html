벡터

벡터(vector)는 Rust에서 같은 타입의 값을 여러 개 저장할 때 쓰는 기본 컬렉션이다. 타입은 Vec<T>이고, 요소들은 메모리에서 서로 이웃한 위치에 저장된다. 그래서 순서가 있고, 인덱스로 접근할 수 있으며, 끝에 값을 추가하거나 제거하는 목록에 잘 맞는다.

let values = vec![1, 2, 3];

벡터는 한 타입의 값만 담는다​

빈 벡터를 만들 때는 요소 타입을 Rust가 추론할 단서가 없으므로 타입을 직접 적어야 한다.

let values: Vec<i32> = Vec::new();

반대로 초깃값이 있으면 vec! 매크로를 통해 타입이 추론된다.

let values = vec![1, 2, 3];

빈 벡터를 만든 뒤 값을 추가해도 타입을 추론할 수 있다.

let mut values = Vec::new();

values.push(5);
values.push(6);
values.push(7);

push는 벡터를 바꾸는 메서드이므로 변수에 mut가 필요하다. 한 번 i32 값을 넣기 시작하면 이 벡터는 Vec<i32>로 굳는다.

인덱싱은 패닉, get은 Option​

벡터 요소를 읽는 대표 방식은 두 가지다.

let values = vec![1, 2, 3, 4, 5];

let third = &values[2];
let maybe_third = values.get(2);

values[2]는 세 번째 요소의 참조를 바로 얻는다. 인덱스가 유효하다는 전제가 강하고, 범위를 벗어나면 패닉이 발생한다.

values.get(2)는 Option<&T>를 반환한다. 인덱스가 없을 수도 있는 상황이면 이쪽이 더 맞다.

match values.get(100) {
    Some(value) => println!("{value}"),
    None => println!("값이 없습니다"),
}

즉 실패가 버그라면 인덱싱을 쓸 수 있고, 실패가 입력이나 상태에 따라 자연스럽게 생길 수 있다면 get으로 처리하는 편이 낫다.

요소 참조를 들고 있는 동안 구조를 바꿀 수 없다​

벡터에서 처음에는 이상해 보이는 규칙이 하나 있다.

let mut values = vec![1, 2, 3, 4, 5];

let first = &values[0];

values.push(6);

println!("{first}");

이 코드는 컴파일되지 않는다. 이미 첫 번째 요소에 대한 불변 참조 first가 살아 있는데, push가 벡터를 가변으로 빌리려 하기 때문이다.

이 규칙은 단순한 형식 제한이 아니다. 벡터는 요소들을 연속된 메모리에 둔다. 끝에 새 값을 추가할 공간이 부족하면 더 큰 메모리 영역을 새로 잡고 기존 요소를 옮길 수 있다. 그러면 first가 가리키던 위치가 더 이상 유효하지 않을 수 있다. Rust는 그 가능성을 대여 규칙으로 막는다.

반복할 때는 벡터 전체를 빌린다​

값을 읽기만 할 때는 불변 참조로 반복한다.

let values = vec![100, 32, 57];

for value in &values {
    println!("{value}");
}

각 요소를 바꿔야 하면 가변 참조로 반복한다.

let mut values = vec![100, 32, 57];

for value in &mut values {
    *value += 50;
}

여기서 value의 타입은 &mut i32이므로 실제 값을 바꾸려면 *value처럼 역참조해야 한다. 이 반복 중에는 벡터 전체가 빌려진 상태이므로, 루프 안에서 요소를 추가하거나 제거하는 식의 구조 변경은 허용되지 않는다.

서로 다른 모양의 값은 열거형으로 묶는다​

벡터는 같은 타입의 값만 담을 수 있다. 여러 종류의 값을 한 목록에 넣어야 한다면, 먼저 그 종류들을 하나의 열거형(enum)으로 묶는다.

enum SpreadsheetCell {
    Int(i32),
    Float(f64),
    Text(String),
}

let row = vec![
    SpreadsheetCell::Int(3),
    SpreadsheetCell::Text(String::from("blue")),
    SpreadsheetCell::Float(10.12),
];

겉보기에는 서로 다른 값이 섞여 있지만, 벡터 입장에서는 모두 SpreadsheetCell 타입이다. 이 방식은 가능한 값의 종류를 컴파일 타임에 알고 있을 때 좋다. 값의 종류가 런타임에 열려 있어야 하면 트레이트 객체 같은 다른 도구가 필요하다.

벡터가 버려지면 요소도 함께 버려진다​

벡터는 스코프를 벗어날 때 자신이 가진 요소들을 함께 정리한다.

{
    let values = vec![1, 2, 3, 4];
}

스코프 밖에서는 values도, 그 안에 있던 요소들도 더 이상 유효하지 않다. 벡터 요소에 대한 참조가 벡터보다 오래 살 수 없는 이유도 여기에 있다.
