Pyodide 빌드 시스템 개선과 WASM SIMD를 통한 성능 최적화에 대한 기여 내용입니다.

빌드 시스템 개선: pyodide clean recipes

Pyodide 빌드 시스템을 사용하면서 소스 파일과 중간 빌드 아티팩트를 정리하는 내장된 방법이 없다는 불편함을 발견했습니다. 개발자들은 수동으로 디렉토리를 삭제해야 했으며, 이는 오류가 발생하기 쉽고 번거로웠습니다.

이를 해결하기 위해 (Issue #218), 두 단계에 걸쳐 포괄적인 정리 솔루션을 구현했습니다:

  1. 핵심 로직 구현 (PR #223): pyodide-build에 도우미 함수를 추가하여 소스 레이아웃과 빌드 디렉토리를 지능적으로 제거하면서 기본적으로 빌드된 휠이 있는 dist 폴더는 보존하도록 했습니다.
  2. CLI 통합 (PR #254): 새로운 CLI 명령인 pyodide clean recipes를 통해 이 기능을 노출했습니다. 이를 통해 사용자는 간단한 플래그로 특정 패키지나 전체 작업 공간을 정리할 수 있게 되어 패키지 유지 관리자의 개발자 경험이 크게 향상되었습니다.

고성능 컴퓨팅: WASM SIMD 활성화

Pyodide의 성능은 브라우저 데이터 과학의 핵심 요소입니다. Emscripten은 C/C++ 코드를 벡터화하여 성능을 최적화하는 WebAssembly SIMD(Single Instruction, Multiple Data)를 지원하지만, Pyodide 내에서 이 생태계가 바로 사용 가능한지는 불분명했습니다. 저는 SIMD 지원을 검증하고 활성화하는 이니셔티브를 주도했습니다 (Issue #5855).

1단계: 검증 (PR #5880)

복잡한 패키지를 최적화하기 전에, Pyodide 환경에서 SIMD 내장 함수가 올바르게 작동하는지 증명해야 했습니다. test-simd라는 새로운 테스트 패키지를 만들어 두 가지 중요한 컴파일 경로를 검증했습니다:

  • wasm_simd128.h 내장 함수의 직접 사용.
  • Emscripten의 SSE/AVX 내장 함수의 WASM SIMD로의 암시적 변환.

2단계: OpenBLAS 최적화 (PR #5960)

검증이 완료된 후, NumPy와 SciPy의 기반 라이브러리인 OpenBLAS라는 실제 사용 사례로 넘어갔습니다. 기존 OpenBLAS 빌드가 SIMD를 활용하지 못하고 있음을 확인했습니다 (Issue #5948).

-msimd128로 컴파일하는 새로운 빌드 레시피인 libopenblas-simd를 구현했습니다. 영향을 정량화하기 위해, Dot Product(sdot) 및 Matrix Multiplication(sgemm)과 같은 주요 연산에 대해 스칼라 대 SIMD 성능을 비교하는 벤치마킹 제품군(test-openblas-simd)을 작성했습니다. 이 작업은 웹에서의 과학 컴퓨팅 성능을 크게 향상시키는 길을 열었습니다.
