Rust에서 변수는 기본적으로 **불변(immutable)** 이다. 즉 `let`으로 값을 한 번 바인딩하면, 같은 변수에 다시 다른 값을 대입할 수 없다. Rust가 이렇게 설계된 이유는 값이 어디서 바뀌는지 추적하기 쉽게 만들고, 안전성과 동시성을 더 잘 보장하기 위해서다.

## 기본 변수

가장 기본적인 변수 선언은 `let`이다.

```rust
fn main() {
    let x = 5;
    println!("x = {x}");
}
```

이 상태에서 `x = 6;`처럼 다시 값을 넣으려고 하면 컴파일 에러가 발생한다.

```rust
fn main() {
    let x = 5;
    println!("x = {x}");
    x = 6;
}
```

에러 메시지는 대략 `cannot assign twice to immutable variable` 형태로 나온다. 즉 "불변 변수에 다시 할당할 수 없다"는 뜻이다.

## 가변 변수

값이 실제로 바뀌어야 하는 변수라면 `mut`를 붙인다.

```rust
fn main() {
    let mut x = 5;
    println!("x = {x}");

    x = 6;
    println!("x = {x}");
}
```

`mut`는 단순히 문법 기능만이 아니라 "이 값은 이후 바뀔 수 있다"는 의도를 코드에 드러내는 역할도 한다.

## 상수

Rust의 상수 (constant)는 `let`이 아니라 `const`로 선언한다.

```rust
const THREE_HOURS_IN_SECONDS: u32 = 60 * 60 * 3;
```

상수는 일반 변수와 비슷하게 보이지만 중요한 차이가 있다.

- 항상 불변이다. `mut`를 붙일 수 없다.
- 타입을 반드시 명시해야 한다.
- 컴파일 타임에 계산 가능한 값만 넣을 수 있다.
- 함수 안뿐 아니라 전역 스코프에서도 선언할 수 있다.
- 이름은 보통 모두 대문자와 `_`를 사용한다.

예를 들어 `const MAX_POINTS: u32 = 100_000;` 같은 식으로 쓰면 프로그램 전체에서 의미 있는 고정값을 표현하기 좋다.

## `let`과 `const` 차이

| 항목 | `let` 변수 | `const` 상수 |
|---|---|---|
| 기본 가변성 | 불변, 필요 시 `mut` 사용 | 항상 불변 |
| 타입 명시 | 보통 생략 가능 | 반드시 필요 |
| 값 계산 시점 | 런타임 값도 가능 | 컴파일 타임 상수 표현식만 가능 |
| 선언 위치 | 일반 스코프 | 일반 스코프 + 전역 스코프 |
| 용도 | 일반적인 값 바인딩 | 프로그램 전반에서 공유하는 고정값 |

## 섀도잉

Rust에서는 같은 이름으로 변수를 다시 선언할 수 있다. 이를 **섀도잉(shadowing)** 이라고 한다.

```rust
fn main() {
    let x = 5;
    let x = x + 1;

    {
        let x = x * 2;
        println!("inner x = {x}");
    }

    println!("outer x = {x}");
}
```

위 코드는 같은 이름 `x`를 다시 쓰지만, 기존 변수를 직접 변경하는 것이 아니라 **새로운 변수 바인딩을 만드는 것**이다.

출력은 다음과 같다.

```text
inner x = 12
outer x = 6
```

안쪽 스코프가 끝나면 안쪽에서 만든 `x`는 사라지고, 바깥 `x`가 다시 보이게 된다.

## 섀도잉이 있는 이유

처음 보면 "굳이 왜 같은 이름으로 다시 선언하게 하지?"라는 생각이 들 수 있다. Rust에서 섀도잉이 유용한 이유는 **같은 대상을 단계적으로 가공할 때 이름은 유지하고 싶지만, 값은 새로 만들고 싶을 때가 많기 때문**이다.

예를 들어 어떤 입력값을 받아서 공백을 제거하고, 그다음 숫자로 파싱한다고 해보자.

```rust
fn main() {
    let input = "  42  ";
    let input = input.trim();
    let input: u32 = input.parse().unwrap();

    println!("input = {input}");
}
```

이 코드는 이름은 계속 `input`이지만 의미는 단계적으로 바뀐다.

- 처음 `input`: 원본 문자열
- 두 번째 `input`: 공백이 제거된 문자열
- 세 번째 `input`: 파싱된 숫자

즉 섀도잉은 "같은 개념을 다음 단계의 값으로 다시 정의한다"는 흐름을 자연스럽게 표현하게 해준다.

만약 이런 상황에서 이름을 전부 따로 지으면 `input_raw`, `input_trimmed`, `input_number`처럼 늘어날 수 있는데, 간단한 변환에서는 오히려 읽기 불편해질 수도 있다.

## 어디에 쓰이나

섀도잉은 보통 다음 같은 경우에 잘 어울린다.

- 같은 값을 단계적으로 정제할 때
- 타입이 바뀌는 변환을 할 때
- 중간 변수 이름을 계속 새로 짓고 싶지 않을 때
- 가공이 끝난 최종 값만 이후 문맥에서 쓰고 싶을 때

예를 들어 문자열 길이로 바꾸는 경우도 섀도잉이 자연스럽다.

```rust
fn main() {
    let spaces = "   ";
    let spaces = spaces.len();

    println!("spaces = {spaces}");
}
```

처음에는 문자열이었지만, 이후에는 "공백 개수"라는 숫자 값으로 의미가 이어진다.

## `mut`와 섀도잉 차이

둘 다 "값이 달라진다"는 점에서는 비슷해 보이지만 의미는 다르다.

- `mut`는 같은 변수의 값을 바꾸는 것이다.
- 섀도잉은 같은 이름으로 새 변수를 다시 만드는 것이다.
- `mut`는 같은 타입 안에서 값을 바꾼다.
- 섀도잉은 타입까지 바꿀 수 있다.

예를 들어 아래 코드는 섀도잉이라 가능하다.

```rust
fn main() {
    let spaces = "   ";
    let spaces = spaces.len();

    println!("spaces = {spaces}");
}
```

처음 `spaces`는 문자열이고, 다음 `spaces`는 숫자다. 같은 이름을 재사용했지만 사실상 새 변수를 만든 것이다.

반면 `mut`로 타입 자체를 바꾸려 하면 에러가 난다.

```rust
fn main() {
    let mut spaces = "   ";
    spaces = spaces.len();
}
```

이 경우에는 처음 변수 타입이 `&str`인데 나중에는 `usize`를 넣으려고 하므로 타입 불일치 에러가 발생한다.

## 언제 무엇을 쓰나

- 값이 실제로 바뀌는 상태라면 `mut`
- 고정된 설정값이나 의미 있는 매직 넘버라면 `const`
- 같은 이름을 유지하면서 값을 가공하거나 타입을 바꾸고 싶다면 섀도잉

예를 들어 카운터처럼 "같은 상태가 계속 변한다"는 의미가 중요하면 `mut`가 더 잘 맞는다.

```rust
fn main() {
    let mut count = 0;

    count += 1;
    count += 1;

    println!("count = {count}");
}
```

반대로 "입력을 점점 정제해서 최종 형태로 만든다"는 흐름이면 섀도잉이 더 읽기 쉽다.

```rust
fn main() {
    let username = "  teddygood  ";
    let username = username.trim();
    let username = username.to_lowercase();

    println!("username = {username}");
}
```

Rust는 기본적으로 불변을 권장하고, 필요할 때만 가변성을 명시하도록 설계되어 있다. 그래서 코드를 읽을 때 "무엇이 바뀌고 무엇이 안 바뀌는지"가 비교적 분명하게 드러난다.
