슬라이스는 컬렉션 전체를 소유하지 않고 그 안의 연속된 일부만 가리키는 참조다. 문자열에서는 `&str`, 배열에서는 `&[T]` 형태로 나타난다. 소유권을 넘기지 않으면서 "이 값의 어느 부분"을 표현할 수 있다는 점이 핵심이다.

## 인덱스만 반환하면 원본과 관계가 끊긴다

문자열의 첫 단어를 찾는 함수를 만든다고 하자. 먼저 떠오르는 방식은 공백 위치를 `usize` 로 반환하는 것이다.

```rust
fn first_word(s: &String) -> usize {
    let bytes = s.as_bytes();

    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return i;
        }
    }

    s.len()
}
```

문제는 반환값 `usize` 가 원래 문자열과 분리된 숫자라는 점이다.

```rust
let mut s = String::from("hello world");
let word = first_word(&s);

s.clear();
```

위 코드에서 `word` 안에는 여전히 `5` 가 남아 있지만, 그 `5` 가 가리키던 문자열 상태는 이미 바뀌어 있다. 숫자는 남아도 의미는 사라진다.

## 문자열 일부를 직접 가리키는 것이 슬라이스다

문자열 슬라이스는 문자열의 일부를 직접 참조한다.

```rust
let s = String::from("hello world");

let hello = &s[0..5];
let world = &s[6..11];
```

`hello` 와 `world` 는 원래 문자열 버퍼를 그대로 가리키고, 시작 위치와 길이 정보만 추가로 가진다. 즉 새 문자열을 복사해 만드는 것이 아니라 기존 버퍼의 일부를 빌려온다.

## 범위 문법은 앞뒤를 생략할 수 있다

```rust
let s = String::from("hello");

let a = &s[0..2];
let b = &s[..2];

let c = &s[3..5];
let d = &s[3..];

let e = &s[0..s.len()];
let f = &s[..];
```

각 쌍은 같은 의미다.

- 앞 인덱스를 생략하면 `0` 부터 시작한다.
- 뒤 인덱스를 생략하면 끝까지 포함한다.
- 둘 다 생략하면 전체 문자열 슬라이스가 된다.

## `first_word` 는 `&str` 를 반환하는 편이 맞다

슬라이스를 사용하면 함수가 원본 문자열과 결과 사이의 관계를 유지할 수 있다.

```rust
fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();

    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }

    &s[..]
}
```

이제 반환값은 단순한 숫자가 아니라 원래 문자열의 일부를 가리키는 참조다.

그래서 아래 코드는 컴파일되지 않는다.

```rust
let mut s = String::from("hello world");
let word = first_word(&s);

s.clear();
println!("{}", word);
```

`word` 가 `s` 를 불변으로 빌리고 있는데, 그 사이에 `clear()` 가 `s` 를 가변으로 빌려 내용을 바꾸려 하기 때문이다. 인덱스 버전에서는 놓쳤던 문제가 슬라이스 버전에서는 컴파일 단계에서 드러난다.

## 매개변수는 `&String` 보다 `&str` 이 더 일반적이다

함수가 문자열 일부를 읽기만 한다면 `&String` 보다 `&str` 을 받는 편이 더 일반적이다.

```rust
fn first_word(s: &str) -> &str {
    let bytes = s.as_bytes();

    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }

    &s[..]
}
```

이렇게 정의하면 세 가지를 모두 받을 수 있다.

```rust
let my_string = String::from("hello world");

let a = first_word(&my_string);
let b = first_word(&my_string[..]);

let my_string_literal = "hello world";
let c = first_word(my_string_literal);
```

`String` 전체를 빌려도 되고, `String` 의 일부분 슬라이스를 넘겨도 되고, 문자열 리터럴도 그대로 넘길 수 있다.

## 문자열 리터럴은 원래 `&str` 이다

```rust
let s = "Hello, world!";
```

여기서 `s` 의 타입은 `String` 이 아니라 `&str` 이다. 문자열 리터럴은 바이너리 안에 저장된 불변 문자열 데이터에 대한 슬라이스로 취급된다.

그래서 문자열 리터럴은 수정할 수 없고, 별도 할당 없이 곧바로 `&str` 인자에 넘길 수 있다.

## 문자열 슬라이스는 UTF-8 경계를 따라야 한다

문자열 슬라이스 인덱스는 바이트 단위다. 그래서 UTF-8 멀티바이트 문자의 중간을 잘라 슬라이스를 만들 수는 없다. 유효한 문자 경계가 아닌 위치로 슬라이스를 만들면 런타임에서 패닉이 발생한다.

즉, 슬라이스는 "문자 인덱스"가 아니라 "바이트 범위"를 자른다고 이해해야 한다.

## 배열 슬라이스도 같은 원리다

슬라이스는 문자열 전용이 아니다.

```rust
let a = [1, 2, 3, 4, 5];
let slice = &a[1..3];

assert_eq!(slice, &[2, 3]);
```

배열 슬라이스 타입은 `&[i32]` 이다. 원본 배열을 소유하지 않고 일부만 참조한다는 점에서 문자열 슬라이스와 같은 원리로 동작한다.

## 정리

- 슬라이스는 컬렉션의 연속된 일부를 가리키는 참조다.
- 문자열 일부를 숫자 인덱스로 따로 들고 있는 것보다 `&str` 로 직접 가리키는 편이 안전하다.
- 문자열을 읽기만 하는 함수는 `&str` 을 받는 쪽이 더 일반적이다.
- 문자열 리터럴은 원래 `&str` 이다.
- 배열도 `&[T]` 슬라이스를 만들 수 있다.
