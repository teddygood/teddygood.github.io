프로그램은 위에서 아래로 실행되지만, 실제 코드에서는 조건에 따라 분기하거나 반복이 필요하다. Rust에서 가장 기본적인 제어 흐름 도구는 `if` 표현식과 반복문인 `loop`, `while`, `for`다.

## `if` 표현식

`if`는 조건에 따라 코드를 분기한다.

```rust
fn main() {
    let number = 3;

    if number < 5 {
        println!("condition was true");
    } else {
        println!("condition was false");
    }
}
```

Rust에서 `if`의 조건은 반드시 `bool`이어야 한다. 다른 언어처럼 정수 `1`, `0`을 참/거짓으로 취급하지 않는다.

```rust
fn main() {
    let number = 3;

    if number {
        println!("number was three");
    }
}
```

위 코드는 컴파일되지 않는다. `number`는 정수이지 `bool`이 아니기 때문이다.

## `else if`

여러 조건을 나눠서 처리하려면 `else if`를 쓴다.

```rust
fn main() {
    let number = 6;

    if number % 4 == 0 {
        println!("number is divisible by 4");
    } else if number % 3 == 0 {
        println!("number is divisible by 3");
    } else if number % 2 == 0 {
        println!("number is divisible by 2");
    } else {
        println!("number is not divisible by 4, 3, or 2");
    }
}
```

조건이 많아질수록 코드가 길어질 수 있으니, 너무 복잡해지면 다른 구조로 빼는 것도 고려할 만하다.

## `if`는 표현식이다

Rust에서 `if`는 단순 제어문이 아니라 **표현식**이기도 하다. 그래서 `let` 오른쪽에 바로 넣을 수 있다.

```rust
fn main() {
    let condition = true;
    let number = if condition { 5 } else { 6 };

    println!("number = {number}");
}
```

단, `if`와 `else`가 만드는 값의 타입은 같아야 한다.

```rust
fn main() {
    let condition = true;
    let number = if condition { 5 } else { "six" };
}
```

이 코드는 한쪽은 정수, 다른 쪽은 문자열이라서 컴파일 에러가 난다.

## `loop`

`loop`는 명시적으로 멈출 때까지 무한 반복한다.

```rust
fn main() {
    loop {
        println!("again!");
    }
}
```

보통은 `break`와 함께 쓴다.

```rust
fn main() {
    let mut count = 0;

    loop {
        count += 1;

        if count == 3 {
            break;
        }
    }

    println!("count = {count}");
}
```

### `break`로 값 반환하기

`loop`는 `break`와 함께 값을 반환할 수도 있다.

```rust
fn main() {
    let mut counter = 0;

    let result = loop {
        counter += 1;

        if counter == 10 {
            break counter * 2;
        }
    };

    println!("result = {result}");
}
```

이 패턴은 어떤 조건이 만족될 때까지 반복한 뒤 그 결과를 바깥으로 꺼내고 싶을 때 유용하다.

### 루프 라벨

중첩 루프에서는 어떤 루프를 빠져나갈지 명시하고 싶을 때가 있다. 이때 라벨을 붙일 수 있다.

```rust
fn main() {
    let mut count = 0;

    'counting_up: loop {
        let mut remaining = 10;

        loop {
            if remaining == 9 {
                break;
            }

            if count == 2 {
                break 'counting_up;
            }

            remaining -= 1;
        }

        count += 1;
    }

    println!("End count = {count}");
}
```

## `while`

조건이 참인 동안 반복하고 싶을 때는 `while`을 사용한다.

```rust
fn main() {
    let mut number = 3;

    while number != 0 {
        println!("{number}!");
        number -= 1;
    }

    println!("LIFTOFF!!!");
}
```

`loop + if + break`로도 같은 구조를 만들 수 있지만, `while`이 더 간결하고 읽기 쉽다.

## `for`

컬렉션 순회에는 `for`가 가장 자연스럽다.

```rust
fn main() {
    let a = [10, 20, 30, 40, 50];

    for element in a {
        println!("the value is: {element}");
    }
}
```

배열 인덱스를 손으로 증가시키는 `while`보다 더 안전하고 간결하다. 길이를 잘못 써서 범위를 벗어나는 실수도 줄일 수 있다.

### 범위 반복

특정 횟수만큼 반복할 때도 `for`를 많이 쓴다.

```rust
fn main() {
    for number in (1..4).rev() {
        println!("{number}!");
    }

    println!("LIFTOFF!!!");
}
```

`1..4`는 1 이상 4 미만 범위를 의미하고, `rev()`는 이를 뒤집는다.

## 무엇을 언제 쓰나

- 단순 조건 분기: `if`
- 값이 필요한 분기: `if` 표현식
- 명시적으로 끊을 때까지 반복: `loop`
- 조건이 참인 동안 반복: `while`
- 컬렉션이나 범위를 순회: `for`

실제로는 컬렉션 순회와 횟수 기반 반복 모두 `for`가 가장 자주 쓰인다.

## 정리

- `if`는 조건 분기이면서 표현식이다.
- `if` 조건은 반드시 `bool`이어야 한다.
- `loop`는 무한 반복이며 `break`로 값을 반환할 수도 있다.
- `while`은 조건 반복에 적합하다.
- `for`는 순회와 반복 횟수 제어에 가장 자주 쓰인다.
