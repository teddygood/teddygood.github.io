구조체(struct)는 서로 관련 있는 값을 하나의 이름 붙은 타입으로 묶는 방법이다. 튜플도 여러 값을 묶을 수 있지만, 구조체는 각 값에 필드(field) 이름을 붙인다. 그래서 데이터의 순서를 기억하는 대신, 코드 안에서 값의 의미를 직접 읽을 수 있다.

```rust
struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}
```

위 정의는 사용자 데이터를 담는 타입의 모양을 만든다. 실제 값을 쓰려면 구조체 이름 뒤에 중괄호를 열고 필드별 값을 채워 인스턴스(instance)를 만든다.

```rust
let user1 = User {
    active: true,
    username: String::from("someusername123"),
    email: String::from("someone@example.com"),
    sign_in_count: 1,
};
```

필드는 점 표기법으로 접근한다.

```rust
let email = &user1.email;
```

인스턴스를 바꾸려면 인스턴스 전체가 `mut`이어야 한다. Rust에서는 특정 필드 하나만 따로 가변으로 표시하지 않는다.

```rust
let mut user1 = User {
    active: true,
    username: String::from("someusername123"),
    email: String::from("someone@example.com"),
    sign_in_count: 1,
};

user1.email = String::from("anotheremail@example.com");
```

## 필드 초기화 축약법

구조체를 만드는 함수에서는 매개변수 이름과 필드 이름이 같아지는 경우가 많다.

```rust
fn build_user(email: String, username: String) -> User {
    User {
        active: true,
        username,
        email,
        sign_in_count: 1,
    }
}
```

이 문법을 **필드 초기화 축약법(field init shorthand)** 이라고 한다. `username: username`, `email: email`처럼 같은 이름을 두 번 쓰지 않고 필드 이름만 적는다. 타입이 달라지는 것이 아니라, 반복을 줄이는 문법이다.

## 구조체 업데이트 문법

기존 인스턴스의 대부분을 유지하고 일부 필드만 바꾸고 싶을 때는 **구조체 업데이트 문법(struct update syntax)** 을 쓴다.

```rust
let user2 = User {
    email: String::from("another@example.com"),
    ..user1
};
```

`..user1`은 위에서 직접 지정하지 않은 나머지 필드를 `user1`에서 가져오겠다는 뜻이다. 이때 문법은 대입처럼 동작하므로, `String`처럼 `Copy`가 아닌 필드는 이동(move)될 수 있다. 위 예에서는 `username`이 `user1`에서 `user2`로 이동한다. 그래서 `user2`를 만든 뒤에는 `user1` 전체를 그대로 사용할 수 없다.

반대로 `active`나 `sign_in_count`처럼 `Copy` 타입인 필드만 가져오는 경우에는 원래 인스턴스의 해당 값이 복사된다. 구조체 업데이트 문법을 쓸 때는 어떤 필드가 이동되고 어떤 필드가 복사되는지 소유권 규칙과 함께 봐야 한다.

## 튜플 구조체와 유사 유닛 구조체

필드 이름까지는 필요 없지만, 튜플 자체에 타입 이름을 붙이고 싶을 때는 **튜플 구조체(tuple struct)** 를 쓴다.

```rust
struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

let black = Color(0, 0, 0);
let origin = Point(0, 0, 0);
```

`Color`와 `Point`는 내부 구성이 모두 `i32` 세 개로 같지만 서로 다른 타입이다. `Color`를 받는 함수에 `Point`를 넘길 수 없다. 이 점이 평범한 튜플과 다르다. 값 자체는 튜플처럼 분해하거나 `.0`, `.1` 같은 인덱스로 접근할 수 있다.

데이터를 전혀 담지 않는 **유사 유닛 구조체(unit-like struct)** 도 만들 수 있다.

```rust
struct AlwaysEqual;

let subject = AlwaysEqual;
```

이런 타입은 데이터를 저장할 필요는 없지만 특정 트레이트를 구현할 타입이 필요할 때 쓴다. 아직 트레이트 구현을 다루기 전이라면, "값은 없지만 타입은 있다"는 정도로 기억해 두면 된다.

## 구조체 필드와 소유권

구조체 필드에도 소유권 규칙이 그대로 적용된다. Rust Book의 `User` 예제는 `username`과 `email`을 `&str`이 아니라 `String`으로 둔다.

```rust
struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}
```

이렇게 하면 `User` 인스턴스가 유효한 동안, 그 안의 문자열 데이터도 인스턴스가 소유한다. 구조체 안에 참조자를 저장할 수도 있지만, 그러려면 참조자가 구조체보다 오래 살아 있음을 라이프타임(lifetime)으로 표현해야 한다.

```rust
struct User {
    active: bool,
    username: &str,
    email: &str,
    sign_in_count: u64,
}
```

위 형태는 라이프타임 매개변수가 없으면 컴파일되지 않는다. 이 단계에서는 구조체가 데이터를 직접 소유해야 하는 필드에 `String` 같은 소유 타입을 쓰는 편이 흐름을 이해하기 쉽다.

## 왜 튜플보다 구조체인가

사각형 넓이를 계산하는 예제를 보면 구조체의 장점이 뚜렷하다. 처음에는 너비와 높이를 따로 넘길 수 있다.

```rust
fn area(width: u32, height: u32) -> u32 {
    width * height
}
```

이 함수는 두 값이 하나의 사각형에 속한다는 사실을 타입으로 표현하지 못한다. 튜플로 묶으면 매개변수는 하나가 된다.

```rust
fn area(dimensions: (u32, u32)) -> u32 {
    dimensions.0 * dimensions.1
}
```

하지만 이제 `0`이 너비이고 `1`이 높이라는 약속을 기억해야 한다. 구조체로 바꾸면 데이터의 관계와 의미가 코드에 남는다.

```rust
struct Rectangle {
    width: u32,
    height: u32,
}

fn area(rectangle: &Rectangle) -> u32 {
    rectangle.width * rectangle.height
}
```

여기서 `area`가 `&Rectangle`을 받는 이유는 넓이를 계산하기 위해 구조체의 소유권을 가져올 필요가 없기 때문이다. 불변 참조로 빌리면 호출한 쪽에서 `rect1`을 계속 사용할 수 있다.

## Debug 출력과 dbg!

사용자가 만든 구조체는 기본적으로 `println!("{}", rect1)`처럼 출력할 수 없다. `{}`는 `Display` 형식을 요구하는데, Rust는 임의의 구조체를 사람이 볼 문자열로 어떻게 표현해야 할지 자동으로 정하지 않는다.

디버깅용 출력이 필요하면 `Debug` 트레이트를 파생한다.

```rust
#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

let rect1 = Rectangle {
    width: 30,
    height: 50,
};

println!("rect1 is {:?}", rect1);
println!("rect1 is {:#?}", rect1);
```

`{:?}`는 한 줄에 가깝게 출력하고, `{:#?}`는 더 읽기 좋은 형태로 출력한다. `dbg!` 매크로도 디버깅에 쓸 수 있다.

```rust
let scale = 2;
let rect1 = Rectangle {
    width: dbg!(30 * scale),
    height: 50,
};

dbg!(&rect1);
```

`dbg!`는 표현식의 값, 파일명, 줄 번호를 표준 에러(stderr)에 출력한다. 표현식의 소유권을 가져왔다가 다시 반환하므로, 구조체 전체를 계속 써야 한다면 `dbg!(&rect1)`처럼 참조를 넘기는 편이 안전하다.

## 메서드와 impl

구조체에 관련된 동작은 함수로 따로 둘 수도 있지만, 구조체의 `impl` 블록 안에 **메서드(method)** 로 정의하면 타입과 동작이 더 가까워진다.

```rust
#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

let rect1 = Rectangle {
    width: 30,
    height: 50,
};

println!("{}", rect1.area());
```

`impl Rectangle` 안에 있는 항목은 `Rectangle` 타입과 연관된다. 메서드의 첫 번째 매개변수는 항상 `self` 계열이다. `&self`는 `self: &Self`의 축약형이고, 여기서 `Self`는 현재 `impl` 대상 타입인 `Rectangle`을 뜻한다.

`area`는 구조체를 읽기만 하므로 `&self`가 맞다. 값을 바꿔야 한다면 `&mut self`를 쓰고, 값을 소비해서 다른 값으로 바꿔야 하는 경우에는 `self`를 쓴다. 다만 `self`만 써서 소유권을 가져오는 메서드는 원본 인스턴스를 이후에 못 쓰게 만들기 때문에 자주 쓰는 형태는 아니다.

메서드는 `self` 뒤에 일반 매개변수를 더 받을 수 있다.

```rust
impl Rectangle {
    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}
```

`rect1.can_hold(&rect2)`처럼 호출하면 `rect1`은 메서드의 수신자(receiver)가 되고, `rect2`는 일반 인수로 들어간다. 이 예제에서도 `other`를 읽기만 하므로 `&Rectangle`을 받는다.

## 자동 참조와 역참조

Rust의 메서드 호출에는 **자동 참조 및 역참조(automatic referencing and dereferencing)** 가 들어간다. 그래서 C나 C++처럼 값 호출에는 `.`, 포인터 호출에는 `->`를 나눠 쓰지 않는다.

```rust
p1.distance(&p2);
(&p1).distance(&p2);
```

메서드의 수신자 타입과 메서드 이름이 정해져 있으면, Rust는 시그니처에 맞게 `&`, `&mut`, `*`를 자동으로 조정한다. 이 덕분에 메서드를 호출할 때는 대체로 값에 점을 찍고 호출하면 된다. 실제 대여 형태는 메서드 시그니처의 `self`, `&self`, `&mut self`가 결정한다.

## 연관 함수

`impl` 블록 안의 모든 함수는 **연관 함수(associated function)** 다. 그중 첫 번째 매개변수로 `self`를 받는 함수가 메서드다. `self`를 받지 않는 연관 함수는 인스턴스 없이 타입 이름으로 호출한다.

```rust
impl Rectangle {
    fn square(size: u32) -> Self {
        Self {
            width: size,
            height: size,
        }
    }
}

let sq = Rectangle::square(3);
```

이런 함수는 생성자처럼 자주 쓰인다. 관례적으로 `new`라는 이름을 많이 쓰지만, `new`는 Rust의 특별한 키워드가 아니다. `Rectangle::square(3)`에서 보이는 `::`는 타입의 네임스페이스 안에 있는 연관 함수를 호출한다는 표시다.

하나의 구조체에는 여러 `impl` 블록을 둘 수 있다.

```rust
impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

impl Rectangle {
    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}
```

이 장의 단순 예제에서는 굳이 나눌 이유가 없지만, 제네릭 타입이나 트레이트 구현을 다룰 때는 여러 `impl` 블록이 자연스럽게 등장한다.
