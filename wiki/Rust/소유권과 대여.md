title: "소유권과 대여"
date: '2026-03-24'

소유권은 값의 메모리를 누가 정리하는지에 대한 규칙이고, 대여는 소유권을 넘기지 않고 값을 읽거나 수정하는 방식이다. Rust는 이 규칙을 컴파일 단계에서 검사한다. 규칙을 어기면 실행 전에 컴파일이 멈춘다.

## 세 가지 소유권 규칙

Rust의 소유권 규칙은 세 줄로 요약된다.

- 각 값에는 소유자가 하나 있다.
- 한 값의 소유자는 동시에 하나만 존재한다.
- 소유자가 스코프를 벗어나면 값은 버려진다.

이 규칙은 스택에 바로 들어가는 단순한 값보다, 힙에 데이터를 두는 타입에서 더 분명하게 드러난다.

## 스택과 힙

정수처럼 크기가 고정된 값은 스택에 저장된다. 반면 `String` 처럼 실행 중 길이가 달라질 수 있는 값은 힙 버퍼를 사용한다.

```rust
let x = 5;
let s = String::from("hello");
```

`x` 는 값 자체가 스택에 들어가고, `s` 는 스택에 포인터, 길이, 용량 같은 메타데이터를 두고 실제 문자열 버퍼는 힙에 둔다.

이 차이 때문에 `String` 은 "누가 이 힙 버퍼를 정리할 것인가"가 중요해진다.

## 스코프를 벗어나면 `drop` 이 호출된다

```rust
{
    let s = String::from("hello");
}
```

위 코드에서 `s` 는 중괄호를 벗어나는 순간 더 이상 유효하지 않다. 이때 Rust는 `drop` 을 호출해 `String` 이 잡고 있던 힙 메모리를 해제한다.

문자열 리터럴은 다르다.

```rust
let s = "hello";
```

문자열 리터럴은 바이너리 안에 들어 있는 불변 데이터에 대한 `&str` 이다. 힙 버퍼를 직접 소유하지 않으므로 `String` 과 정리 방식이 다르다.

## 대입하면 복사가 아니라 이동일 수 있다

정수는 대입해도 그대로 둘 다 쓸 수 있다.

```rust
let x = 5;
let y = x;

println!("x = {}, y = {}", x, y);
```

하지만 `String` 은 다르다.

```rust
let s1 = String::from("hello");
let s2 = s1;

println!("{}", s1);
```

이 코드는 컴파일되지 않는다. `s1` 의 소유권이 `s2` 로 이동했기 때문이다.

여기서 힙 버퍼 내용 전체를 깊은 복사하는 것이 아니라, 스택에 들어 있는 포인터, 길이, 용량 정보만 옮긴 뒤 원래 변수 `s1` 을 더 이상 사용하지 못하게 만든다. 이렇게 해야 같은 힙 버퍼를 두 변수가 동시에 해제하는 상황을 막을 수 있다.

## 깊은 복사가 필요하면 `clone`

같은 문자열 데이터를 독립적으로 두 개 가져야 한다면 `clone` 을 쓴다.

```rust
let s1 = String::from("hello");
let s2 = s1.clone();

println!("s1 = {}, s2 = {}", s1, s2);
```

이 경우에는 힙 버퍼까지 새로 복사한다. 그래서 `clone` 은 단순 대입과 달리 비용이 들어갈 수 있다.

## `Copy` 타입은 이동되지 않는다

모든 타입이 이동되는 것은 아니다. `i32`, `bool`, `char` 같은 타입은 `Copy` 를 구현한다.

- 모든 정수형
- 모든 부동 소수점형
- `bool`
- `char`
- `Copy` 타입만으로 이루어진 튜플

이런 값은 대입이나 함수 인자 전달 시 값이 그대로 복사되므로 원래 변수도 계속 유효하다.

반대로 `String` 처럼 힙 버퍼를 소유하는 타입은 기본적으로 `Copy` 가 아니다.

## 함수에 넘길 때도 이동이 일어난다

대입에서 이동이 일어난다면 함수 인자 전달에서도 같은 규칙이 적용된다.

```rust
fn takes_ownership(s: String) {
    println!("{}", s);
}

fn makes_copy(x: i32) {
    println!("{}", x);
}

fn main() {
    let s = String::from("hello");
    takes_ownership(s);

    let x = 5;
    makes_copy(x);

    println!("{}", x);
}
```

여기서 `s` 는 함수 호출과 함께 이동되므로 이후에 사용할 수 없다. 반면 `x` 는 `Copy` 타입이라 호출 이후에도 계속 쓸 수 있다.

## 반환값도 소유권을 이동시킨다

함수가 값을 반환할 때도 소유권은 함께 이동한다.

```rust
fn gives_ownership() -> String {
    String::from("yours")
}

fn takes_and_gives_back(s: String) -> String {
    s
}

fn main() {
    let s1 = gives_ownership();
    let s2 = String::from("hello");
    let s3 = takes_and_gives_back(s2);
}
```

`gives_ownership` 의 반환값은 `s1` 이 소유하고, `s2` 는 함수 인자로 이동한 뒤 반환값으로 다시 `s3` 에 들어간다.

이런 흐름은 규칙상 자연스럽지만, 단순히 길이만 계산하려고 값을 넘겼다가 다시 반환받는 식은 번거롭다. 그래서 참조가 필요해진다.

## 참조는 소유권 없이 값을 본다

값을 넘기지 않고 읽기만 하려면 참조를 만든다.

```rust
fn calculate_length(s: &String) -> usize {
    s.len()
}

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);

    println!("The length of '{}' is {}.", s1, len);
}
```

`&s1` 은 `s1` 을 빌려온 참조다. 참조는 값을 소유하지 않으므로 함수가 끝나도 원래 값은 호출자 쪽에 그대로 남아 있다.

이렇게 참조를 만드는 것을 대여라고 부른다.

## 가변 대여는 `&mut`

참조로 읽기만 하는 것이 아니라 값을 바꾸려면 가변 참조가 필요하다.

```rust
fn change(s: &mut String) {
    s.push_str(", world");
}

fn main() {
    let mut s = String::from("hello");
    change(&mut s);
}
```

여기서는 두 군데 모두 `mut` 가 필요하다.

- 원래 값이 가변이어야 한다.
- 함수가 `&mut String` 을 받아야 한다.

## 한 시점에는 하나의 가변 참조만 가능하다

```rust
let mut s = String::from("hello");

let r1 = &mut s;
let r2 = &mut s;
```

이 코드는 컴파일되지 않는다. 같은 값에 대해 동시에 둘 이상의 가변 참조를 만들 수 없기 때문이다.

이 규칙은 값 수정이 어디서 일어나는지를 한 시점에 하나로 제한한다.

## 불변 참조와 가변 참조는 동시에 공존할 수 없다

```rust
let mut s = String::from("hello");

let r1 = &s;
let r2 = &s;
let r3 = &mut s;
```

이 코드도 컴파일되지 않는다. `r1`, `r2` 는 값을 읽기 전용으로 빌리고 있는데, 같은 시점에 `r3` 가 값을 수정할 수 있게 되면 읽기 측이 전제하는 상태가 깨지기 때문이다.

반대로 불변 참조만 여러 개 만드는 것은 가능하다.

```rust
let s = String::from("hello");

let r1 = &s;
let r2 = &s;
println!("{}, {}", r1, r2);
```

## 참조의 유효 범위는 마지막 사용까지다

참조는 변수가 선언된 줄부터 무조건 스코프 끝까지 살아 있는 것이 아니라, 마지막으로 사용된 지점까지 유효한 것으로 계산된다.

```rust
let mut s = String::from("hello");

let r1 = &s;
let r2 = &s;
println!("{} and {}", r1, r2);

let r3 = &mut s;
println!("{}", r3);
```

이 코드는 컴파일된다. `r1`, `r2` 는 `println!` 이후 더 이상 사용되지 않으므로, 그 다음 줄의 `r3` 와 유효 범위가 겹치지 않는 것으로 처리된다.

## 댕글링 참조는 허용되지 않는다

```rust
fn dangle() -> &String {
    let s = String::from("hello");
    &s
}
```

이 함수는 컴파일되지 않는다. `s` 는 함수가 끝날 때 버려지는데, 그 참조를 반환하면 이미 해제된 메모리를 가리키는 참조가 생기기 때문이다.

이럴 때는 참조가 아니라 값을 직접 반환해야 한다.

```rust
fn no_dangle() -> String {
    let s = String::from("hello");
    s
}
```

## 정리

- `String` 처럼 힙 데이터를 소유하는 타입은 대입과 함수 호출에서 이동이 일어날 수 있다.
- 깊은 복사가 필요할 때만 `clone` 을 쓴다.
- `Copy` 타입은 대입과 호출 시 그대로 복사된다.
- 참조는 소유권 없이 값을 본다.
- 한 시점에는 여러 불변 참조 또는 하나의 가변 참조만 허용된다.
- 참조가 살아 있는 동안 원본 값은 그 규칙에 맞게만 접근할 수 있다.
- 이미 버려질 값을 가리키는 참조는 만들 수 없다.
