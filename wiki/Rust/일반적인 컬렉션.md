Rust의 컬렉션(collection)은 값을 하나만 들고 있는 타입이 아니라 여러 값을 모아 관리하는 타입이다. 배열이나 튜플처럼 크기와 모양이 비교적 고정된 타입과 달리, 표준 라이브러리 컬렉션은 보통 힙에 데이터를 두고 실행 중에 크기가 늘거나 줄 수 있다.

Rust Book 8장에서 먼저 잡아야 할 축은 세 가지다.

- `Vec<T>`는 같은 타입의 값을 순서대로, 연속된 메모리 공간에 저장한다.
- `String`은 UTF-8 바이트를 소유하는 문자열 컬렉션이다.
- `HashMap<K, V>`는 키를 통해 값을 찾는 매핑 구조다.

이 셋은 모두 "값을 담는 구조"지만, 읽는 기준이 다르다. 벡터는 위치가 기준이고, 문자열은 바이트와 UTF-8 경계가 기준이며, 해시맵은 키가 기준이다. 그래서 같은 "조회"라도 실패 처리와 비용 모델이 다르게 나타난다.

## `Vec<T>`는 순서 있는 목록이다

벡터(vector)는 같은 타입의 값을 여러 개 담을 때 가장 먼저 떠올릴 수 있는 컬렉션이다. 파일의 줄 목록, 장바구니 가격 목록, 계산 결과의 나열처럼 순서가 있고 인덱스로 접근할 일이 있는 데이터에 맞다.

```rust
let mut values = Vec::new();

values.push(10);
values.push(20);
values.push(30);
```

초깃값이 있으면 `vec!` 매크로를 쓰는 편이 간단하다.

```rust
let values = vec![10, 20, 30];
```

벡터에서 특히 조심할 부분은 "요소 참조를 들고 있는 동안 벡터 구조를 바꿀 수 없다"는 점이다. `push`는 새 공간으로 재할당을 일으킬 수 있고, 그러면 기존 요소를 가리키던 참조가 더 이상 안전하지 않다. Rust는 이 가능성을 컴파일 단계에서 막는다.

## `String`은 UTF-8 바이트 컬렉션이다

`String`은 단순히 "문자 배열"이 아니다. 표준 라이브러리가 제공하는, 커질 수 있고 수정 가능한 UTF-8 문자열이다. 내부적으로는 바이트 벡터에 가까운 구조를 가지지만, 그 바이트가 유효한 UTF-8이라는 보장을 함께 갖는다.

```rust
let mut text = String::from("hello");

text.push_str(", world");
text.push('!');
```

문자열에서 가장 큰 함정은 인덱싱이다. `text[0]`처럼 숫자 하나로 접근하면 "첫 번째 바이트", "첫 번째 유니코드 스칼라 값", "사용자가 보는 첫 번째 글자" 중 무엇을 뜻하는지 명확하지 않다. 그래서 Rust는 `String`의 숫자 인덱싱을 지원하지 않는다.

필요한 관점을 명시해야 한다.

```rust
for c in "Зд".chars() {
    println!("{c}");
}

for b in "Зд".bytes() {
    println!("{b}");
}
```

문자 단위가 필요하면 `chars`, 바이트 단위가 필요하면 `bytes`를 쓴다. 사용자가 보는 글자 단위인 문자소 클러스터(grapheme cluster)가 필요하면 표준 라이브러리만으로는 부족할 수 있다.

## `HashMap<K, V>`는 키로 값을 찾는다

해시맵(hash map)은 순서나 위치보다 키가 중요한 데이터에 쓴다. 예를 들어 팀 이름에서 점수를 찾거나, 단어에서 등장 횟수를 찾을 때 자연스럽다.

```rust
use std::collections::HashMap;

let mut scores = HashMap::new();

scores.insert(String::from("Blue"), 10);
scores.insert(String::from("Yellow"), 50);
```

조회 결과는 실패할 수 있으므로 `get`은 `Option<&V>`를 반환한다.

```rust
let team = String::from("Blue");
let score = scores.get(&team).copied().unwrap_or(0);
```

해시맵에서 자주 쓰는 업데이트 패턴은 `entry`다. 키가 없을 때만 기본값을 넣고, 그 값을 수정하는 흐름을 대여 검사기와 잘 맞게 표현한다.

```rust
let count = scores.entry(String::from("Blue")).or_insert(0);
*count += 1;
```

## 어떤 컬렉션을 고를까

목록의 순서와 인덱스가 중요하면 `Vec<T>`가 먼저다. 텍스트를 소유하고 수정해야 하면 `String`이다. 어떤 이름, 식별자, 키로 값을 찾아야 하면 `HashMap<K, V>`가 맞다.

세 타입 모두 소유권과 대여 규칙을 그대로 따른다. 벡터에 값을 넣으면 벡터가 그 값을 소유하고, 문자열 결합은 어떤 문자열의 소유권을 소비할 수 있으며, 해시맵에 `String` 키와 값을 넣으면 해시맵이 그 값을 소유한다. 컬렉션 API를 읽을 때는 "이 메서드가 값을 복사하는가, 빌리는가, 이동시키는가"를 같이 봐야 한다.
