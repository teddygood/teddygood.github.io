우연히 Windows의 C드라이브를 정리하다가 `hiberfil.sys`라는 파일이 용량을 꽤 먹고 있었다. 알고 보니 하이버네이션(hibernation)이라는 기술과 관련된 것이었고 그걸 가볍게 정리해본다.

하이버네이션은 Windows가 메모리 상태를 디스크의 `hiberfil.sys`에 저장한 뒤 사실상 꺼지는 전원 상태다. 빠른 시작(Fast Startup)은 이 하이버네이션 인프라를 이용해서 사용자 세션은 종료하고, 커널 세션(kernel session)만 저장했다가 다음 부팅 때 다시 읽어 오는 방식이다.

둘은 비슷해 보이지만 목적이 다르다. 하이버네이션은 **작업 상태 보존**이 목적이고, 빠른 시작은 **부팅 경로 단축**이 목적이다.

## 하이버네이션이란 무엇인가

Windows 전원 모델에서 하이버네이션은 보통 `S4` 상태로 설명된다. 절전(sleep)처럼 RAM에 전원을 유지하지 않고, 메모리 내용을 디스크에 저장한 뒤 시스템 전력을 매우 낮은 수준으로 내린다.

이 방식의 장점은 전력을 거의 쓰지 않으면서도 이전 작업 상태를 복원할 수 있다는 점이다. 대신 RAM을 그대로 유지하는 절전보다 복귀 속도는 보통 더 느리다.

## 절전, 하이버네이션, 빠른 시작 차이

| 항목                  | 핵심 아이디어                    | 상태 저장 위치       | 주된 목적           |
| ------------------- | -------------------------- | -------------- | --------------- |
| 절전(Sleep)           | RAM을 유지한 채 저전력 상태로 들어감     | RAM            | 빠른 복귀           |
| 하이버네이션(Hibernation) | 메모리 내용을 디스크에 저장하고 전원을 거의 끔 | `hiberfil.sys` | 상태 보존과 낮은 전력 사용 |
| 빠른 시작(Fast Startup) | 사용자 세션은 종료하고 커널만 저장        | `hiberfil.sys` | 종료 후 다음 부팅 가속   |

빠른 시작은 "완전 종료처럼 보이는 하이브리드 종료"에 가깝다. 즉 종료를 눌렀다고 해서 항상 커널까지 완전히 새로 시작되는 것은 아니다. 그래서 Windows는 일부 업데이트나 드라이버 반영 상황에서 `다시 시작`을 더 확실한 경로로 취급한다.

## `hiberfil.sys`가 하는 일

`hiberfil.sys`는 하이버네이션과 빠른 시작이 공통으로 사용하는 시스템 파일이다. 이 파일이 있어야 Windows가 메모리 또는 커널 상태를 디스크에 저장한 뒤 다시 읽어 올 수 있다.

중요한 점은 이 파일이 단순 캐시가 아니라, 전원 상태 전환 자체를 가능하게 하는 파일이라는 점이다. 그래서:

- 하이버네이션을 끄면 `hiberfil.sys`가 제거된다.
- 빠른 시작도 함께 꺼지는 경우가 일반적이다.
- 디스크 공간이 부족할 때는 이 파일이 체감상 꽤 큰 공간을 차지할 수 있다.

## 데스크톱에서 꼭 필요한가

노트북에서는 배터리 절약과 작업 상태 보존 때문에 하이버네이션의 가치가 크다. 반면 데스크톱에서는 필요성이 보통 더 낮다.

데스크톱에서 하이버네이션이 유용한 경우는 다음과 같다.

- 정전이나 멀티탭 전원 차단 위험이 있다.
- 오랜 뒤에도 열어 둔 프로그램 상태를 그대로 복원하고 싶다.
- 절전보다 더 낮은 전력 사용이 필요하다.

반대로 다음 패턴이면 굳이 사용하지 않아도 된다.

- 평소 `종료` 위주로 사용한다.
- 항상 전원 연결 상태다.
- 저장 공간이 부족하다.
- 빠른 시작이 주는 몇 초의 차이보다 디스크 공간 회수가 더 중요하다.

즉 데스크톱에서는 하이버네이션이 "기본 필수 기능"이라기보다, 필요할 때만 유지할 옵션에 가깝다.

## WSL과는 무슨 관계가 있나

혹시 내가 사용하는 WSL과 관련이 있을까 찾아봤지만 관계는 없어 보인다.

굳이 SL 때문에 하이버네이션이나 빠른 시작을 꼭 유지해야 하는 것은 아니다. 빠른 시작의 직접 목적은 WSL 세션 유지가 아니라, Windows 커널 초기화 비용을 줄여 다음 부팅을 빠르게 만드는 데 있다.

그래서 평소 시스템을 `종료`하는 사용 패턴이라면, WSL을 쓴다는 이유만으로 하이버네이션을 유지해야 할 운영상 이유는 보통 없다.

## 빠른 시작은 얼마나 빠른가

빠른 시작은 뭐 얼마나 빠르길래 빠른 시작일까?

Microsoft는 모든 PC에 대해 고정된 개선폭을 약속하지 않는다. 빠른 시작의 효과는 다음 요소에 크게 좌우된다.

- 펌웨어 POST 시간
- 저장장치 속도
- 시작 프로그램 수
- 드라이버 초기화 비용
- 백신과 백그라운드 서비스

따라서 최신 SSD/NVMe 기반 데스크톱에서는 체감 차이가 크지 않을 수 있고, 느린 저장장치나 부팅 경로가 무거운 시스템에서는 차이가 더 커질 수 있다. 핵심은 "항상 엄청 빨라진다"가 아니라, **full boot를 줄이는 구조적 최적화**라는 점이다.

## 하이버네이션을 어떻게 끄는가?

관리자 권한으로 아래 명령을 실행하면 된다.

```powershell
powercfg /h off
```

이 경우 일반적으로 아래와 같은 변화가 생긴다.

- 하이버네이션 비활성화
- 빠른 시작 비활성화
- `hiberfil.sys` 제거
- 디스크 공간 회수

다시 켜고 싶으면 다음 명령을 쓴다.

```powershell
powercfg /h on
```

## Takeaway

하이버네이션은 작업 상태를 디스크에 저장하는 `S4` 기반 기능이고, 빠른 시작은 그 인프라를 이용해 커널만 저장하는 부팅 최적화다. 노트북에서는 가치가 크지만, 데스크톱에서는 사용 패턴과 디스크 여유 공간을 보고 유지 여부를 판단하는 편이 더 합리적이다.
